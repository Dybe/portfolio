          A/B  TESTING

A/B testing, for websites, is when you show two different variations of the same webpage to different visitors at the same time in order to determine what variation of the website works better for the desired user behavior.

In this project, I performed A/B Testing for an online cactus shop. I wanted to determine if very small changes to the user interface would cause users to behave differently.

For one variant of the website (let’s call this variant A), I increased the size of  “Add to Cart ” buttons. For the other variant, (let’s call it variant B), I increased the size of cacti images.

Metrics & Hypotheses

I monitored two metrics for the test.  Time to Completion (TOC), which is the time it took a visitor to complete their order before clicking checkout; and the proportion of users that returned to the page, Return Rate (RR).

I came up with null and alternative hypotheses for these metrics then collected data to test the hypotheses.

Hypotheses on Time to Completion

I expect a larger size of “Add to Cart” buttons will make it easier for users to add their desired cacti to the cart.

Null Hypothesis: Increasing the size of “Add to Cart” buttons has no effect in the time it takes a user to complete an order, Variants A and B will have similar TOC.

Alternative Hypothesis: Increasing the size of “Add to Cart” buttons will reduce the time it takes a user to complete an order, Variant A will have a lower TOC than B.

Hypotheses on Return Rate

I expect a larger picture size for the cacti to provide an overall better user experience by creating a more visually appealing UI and therefore encourage users to return to the site. Larger photos will also ensure that users have a better view of the cacti they buy which could encourage users to return to the size for more cacti if they got exactly what they desired from their previous orders.

Null Hypothesis: Increasing the size of cacti photos will have no impact on the return rate, Variant A and B will have similar RR.

Alternative Hypothesis: Increasing the size of cacti photos will increase return rate, Variant B will have a higher RR than A.

Data Analysis and Conclusions

I performed two tests on the data in order to test my hypotheses. I first performed a Two-Sample T-Test  and a Chi-square Test for homogeneity with a significance level of 0.5 to test my hypotheses on Time to Completion and Return Rate respectively.

From my T-test, the p-value was 0.283. Since this value is greater than the significance level, my results are statistically insignificant. I cannot reject my null hypothesis on Time to Completion: Increasing the size of “Add to Cart” buttons has no effect in the time it takes a user to complete an order, Variants A and B will have similar TOC. Variant A,therefore, despite having larger buttons, is no better than Variant B.  

Similarly, the Chi-square Test produced a p-value of 0.568 which also by being greater than the significance level shows that my results for Return Rate are statistically insignificant. I cannot reject my null hypothesis on Return Rate: Increasing the size of cacti photos will have no impact on the return rate, Variant A and B will have similar RR. Variant B did not perform better than A even though it had bigger cacti photos.

                

                                

Limitations & Learnings

  • My dataset was very small. I would have liked to see how the variants perform with significantly more user sessions ( > 1000)  and from users of a wider range of backgrounds (e.g age etc) than my classmates, perhaps the results would have been different.

  • My changes were somewhat random, they were not based on specific user needs/ feedback. It would have been interesting to see how changes based on user feedback perform.

  • Overall, I am proud of my work! I got to solidify my understanding of A/B testing, an important and popular tool in measuring the impact of small feature changes on user behavior.
